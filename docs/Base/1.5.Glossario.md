# 1.5. Glossário

## 1.5.1 Introdução

O Glossário é um artefato que lista e define os termos técnicos, acrônimos, abreviações e jargões específicos utilizados no contexto de um projeto, ele serve como referência centralizada para garantir que todos os stakeholders tenham uma compreensão comum e consistente da linguagem utilizada no projeto.

## 1.5.2 Glossário

| Termo                 | Significado                                                                                                                                                                                                                                                                                                                                                  |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 5W2H                  | É uma ferramenta de gestão utilizada para a elaboração de planos de ação de forma estruturada e detalhada a partir de sete perguntas que, em inglês cinco começam com W ("O que?", "Por que?", "Onde?", "Quem vai participar?", "Quando?"), e duas começando em H ("Como?" e "Quanto vai custar?").                                                          |
| Abordagens            | Conjunto de atividades realizadas de forma mais simples, menos diretrizadas e mais flexíveis do que metodologias e processos.                                                                                                                                                                                                                                |
| Acessibilidade        | Capacidade de o usuário de acessar o sistema para interagir com ele, sem que a interface imponha obstáculos.                                                                                                                                                                                                                                                 |
| Brainstorm            | Atividade que busca levantar de forma livre um conjunto grande e abrangente de opiniões dos participantes em torno de um tema, normalmente sendo em volta dos problemas e necessidades do cliente.                                                                                                                                                           |
| BPMN                  | Trata-se de uma notação a qual é comumente utilizada para especificação de processos de desenvolvimento, essa notação permite especificar atividades de vários tipos, retroalimentação (loops), temporalidade, participantes, dentre outros aspectos inerentes à documentação de um processo de desenvolvimento de software.                                 |
| Console               | Equipamento eletrônico semelhante a um computador cujo principal propósito é a execução de videogames.                                                                                                                                                                                                                                                       |
| Design                | Termo amplo utilizado para definir o desenho de um aspecto do software a partir de uma série de requisitos, uma maneira de construir um aspecto do software que seja funcional, eficiente e amigável ao usuário, o design pode ser separado em Design de Interfaces, Design de Arquitetura, entre outros.                                                    |
| Design Sprint         | Primeira sprint realizada em projetos ágeis que trabalha o problema do cliente e desenvolve soluções e testes de ideias por meio de prototipação e outros artefatos, normalmente no período de 5 dias intensos.                                                                                                                                              |
| Diagrama Causa-Efeito | Abordagem que desenvolve um diagrama em formato de espinha de peixe para descrever claramente o problema ao definir os fatores que levam ao problema raiz.                                                                                                                                                                                                   |
| E-commerce            | É uma plataforma de comércio digital, onde a disponibilização dos produtos e o pagamento são realizados de forma online para que o produto seja enviado para o cliente, seja um produto físico ou digital.                                                                                                                                                   |
| Elicitação            | Período do planejamento de um produto onde se entende o problema, identifica, coleta e refina os requisitos do produto a ser desenvolvido.                                                                                                                                                                                                                   |
| Feature               | Parte de um sistema que realiza uma função ou um conjunto de funções, são componentes essenciais que contribuem com a usabilidade e valor de um produto de software.                                                                                                                                                                                         |
| GitPages              | Página web que hospeda toda a documentação, diretrizes e informações a respeito do projeto e seu escopo.                                                                                                                                                                                                                                                     |
| Interface             | Porção do sistema com a qual o usuário mantém contato físico (motor ou perceptivo) ou conceitual durante a interação.                                                                                                                                                                                                                                        |
| Kanban                | Ferramenta de trabalho usada para implementar desenvolvimento de software ágil que enfatiza a transparência do trabalho e a capacidade de comunicação em tempo real.                                                                                                                                                                                         |
| Mapa Mental           | Representação da percepção e arredores que uma pessoa cria a partir de uma ideia central para ajudar a entender a ambientação dessa ideia.                                                                                                                                                                                                                   |
| Metodologias          | Abordagem sistemática e lógica por trás de um projeto de pesquisa utilizada no desenvolvimento de projetos.                                                                                                                                                                                                                                                  |
| Modelagem             | Processo de criação de uma representação visual simplificada de um sistema de software e seus componentes a partir do uso de elementos gráficos e textuais.                                                                                                                                                                                                  |
| OPENUP                | Processo de desenvolvimento que procura otimizar e agilizar o Processo Unificado.                                                                                                                                                                                                                                                                            |
| Pareamento            | Ação de desenvolver um produto de software por duas pessoas de forma síncrona e normalmente na mesma máquina, com o objetivo de tornar o desenvolvimento mais eficiente.                                                                                                                                                                                     |
| Processos             | Conjunto de atividades e orientações rigorosas que costumam estabelecer porcentagens de esforços da equipe para cada fase do processo de desenvolvimento.                                                                                                                                                                                                    |
| Protótipo             | Representação visual criada para avaliação da proposta de solução do problema junto ao usuário.                                                                                                                                                                                                                                                              |
| Sprint Review         | Evento com o propósito de analisar e inspecionar o resultado da Sprint para determinar futuras adatapções na realização dos outros rituais do Scrum.                                                                                                                                                                                                         |
| Rich Picture          | Representação visual usada para explorar, reconhecer e definir uma situação ou problema, similar a um mapa mental, o Rich Picture auxilia equipes a compreender o problema ou situação que devem solucionar, essa representação encoraja o uso de desenhos, símbolos e imagens para descrever melhor a situação.                                             |
| SCRUM                 | Metodologia ágil de organização de equipes, possui diversos eventos para organizar o trabalho necessário para realizar uma tarefa ou criar um produto, a metodologia traz uma maior ênfase em no valor do produto e na transparência.                                                                                                                        |
| Software              | É a coleção de programas, funções, dados e informações necessárias para que o computador ou aparelho eletrônico execute uma tarefa específica.                                                                                                                                                                                                               |
| Sprint Planning       | Evento crucial para a execução do SCRUM, é uma reunião realizada no inicio de cada sprint e seu objetivo é realizar o planejamento para a próxima sprint, o que inclui a escolha dos requisitos que serão trabalhados e como será feito esse trabalho.                                                                                                       |
| Stand Up Meeting      | É um tipo de reunião comum em um contexto de desenvolvimento regido pelo XP, é uma reunião diária feita pelo time de desenvolvimento para comunicar progresso, identificar problemas e promover a participação da equipe, essas reuniões acontecem rapidamente e enquanto todos estão em pé, para reforçar que a reunião é curta.                            |
| Status                | Termo que representa o estado atual de algo, pode se referir ao estado de disponibilidade de um serviço ou a finalização de uma atividade, produto ou similar.                                                                                                                                                                                               |
| Storyboard            | Técnica visual utilizada no desenvolvimento de softwares para demonstrar um processo ou produto a ser desenvolvido, normalmente envolve uma história de alguém que possui um problema e como o produto proposto irá solucionar.                                                                                                                              |
| Usuários              | Termo utilizado para se referir à pessoa que utiliza a aplicação, normalmente vinculado ao papel do cliente, este possui diferentes objetivos com a aplicação e seus desejos devem ser previstos pela equipe de desenvolvimento para a produção de um produto de qualidade.                                                                                  |
| Videogame             | Software com objetivo de trazer entretenimento ao seus usuários, são produtos altamente interativos que envolvem histórias, desafios ou qualquer aspecto que seja divertido, inovador ou empolgante.                                                                                                                                                         |
| Wiki                  | Plataforma online de compartilhamento de informações, permite a edição de seu conteúdo por vários usuários e possui como objetivo agrupar uma quantidade de informação correlacionada, este termo será utilizado para se referir à está página de documentação do projeto.                                                                                   |
| Wireframes            | É uma representação visual de um interface, o objetivo é servir como um modelo inicial dessas interfaces para projetar e identificar funcionalidades e conteúdos desta.                                                                                                                                                                                      |
| XP                    | É uma metodologia ágil de desenvolvimento de software, cuja as principais ênfases são entregas rápidas e frequentes em ciclos de desenvolvimento curto com o objetivo de melhorar a produtividade e incluir ajustes em requisitos sem prejudicar a produção do software, os cinco valores do XP são Comunicação, Simplicidade, Feedback, Coragem e Respeito. |
